# 语法

正则的语法虽然符号多，但逻辑极其自洽。它本质上是一套**“微型编程语言”**，有自己的“数据类型”和“运算规则”。

## 字符表示法（匹配“什么”）

这是最基础的原子单位。

| 语法               | 含义                                                 | 雷区提醒                                                     |
| :----------------- | :--------------------------------------------------- | :----------------------------------------------------------- |
| **普通字符**       | `abc` 匹配字面量的 `abc`                             | 绝大多数字母数字都是字面量，但 `\` 开头的是转义符。          |
| **`\d`**           | 匹配任意**数字**，等价于 `[0-9]`                     | **注意**：在 Python 中默认不匹配 Unicode 数字（如 １２），需加 `re.ASCII` 标志。 |
| **`\w`**           | 匹配**单词字符**，等价于 `[A-Za-z0-9_]`              | 默认不匹配中文。若想匹配中文，请用 `[\u4e00-\u9fa5]` 或 `\p{Han}`（需引擎支持）。 |
| **`\s`**           | 匹配**空白字符**，包括空格、制表符 `\t`、换行符 `\n` | 不匹配全角空格（中文空格）。                                 |
| **`.`**            | 匹配**除换行符 `\n` 外的任意字符**                   | 最危险的符号！若想匹配真正的点号 `.`，必须转义为 `\.`。      |
| **`\D` `\W` `\S`** | 分别匹配**非**数字、非单词、非空白                   | 大写表示取反，逻辑清晰。                                     |



## 字符组 `[ ]`（匹配“其中一个”）

用来明确指定“在这一堆字符里，只要命中一个就算成功”。

- **基本用法**：`[aeiou]` 匹配任意一个英文元音字母。
- **范围表示**：`[0-9A-F]` 匹配十六进制数字（0-9或A-F）。
- **关键细节（雷区）**：
  - **`^` 的位置**：`[^abc]` 表示匹配**除了** a、b、c 之外的任意字符（取反）。如果 `^` 不在开头，如 `[a^bc]`，它就是一个普通字符 `^`。
  - **`-` 的位置**：`[a-z]` 是范围。如果你想匹配字面量的横杠 `-`，必须把它放在**开头或结尾**，如 `[-abc]` 或 `[abc-]`，否则会报错或产生非预期范围。

## 量词 `{}`（匹配“多少次”）

用来控制前面那个原子重复出现的次数。

| 语法    | 含义                                       | 等价写法 |
| :------ | :----------------------------------------- | :------- |
| `{n}`   | 必须出现 **恰好** n 次                     | 无       |
| `{n,}`  | 出现 **至少** n 次                         | 无       |
| `{n,m}` | 出现 **n 到 m** 次                         | 无       |
| `*`     | 出现 **0 次或无数次**（可有可无）          | `{0,}`   |
| `+`     | 出现 **1 次或无数次**（至少一次）          | `{1,}`   |
| `?`     | 出现 **0 次或 1 次**（要么没有，要么一次） | `{0,1}`  |



## 贪婪与懒惰（量词的“性格”）

这是新手最懵圈的地方。**同样的量词，加不加 `?`，匹配结果天差地别。**

- **贪婪模式（默认）**：在匹配成功的前提下，**尽可能多地**匹配字符。
  - 文本：`"<p>段落1</p><p>段落2</p>"`
  - 正则：`/<p>.*<\/p>/` 
  - **结果**：匹配**整个** `"<p>段落1</p><p>段落2</p>"`。因为 `.*` 会一直吃到最后一个 `</p>` 才罢休。

- **懒惰模式（加 `?`）**：在匹配成功的前提下，**尽可能少地**匹配字符。
  - 正则：`/<p>.*?<\/p>/`
  - **结果**：分别匹配到 `"<p>段落1</p>"` 和 `"<p>段落2</p>"`。遇到第一个 `</p>` 就停止。

- **常用懒惰组合**：`*?`、`+?`、`{n,m}?`。

## 位置锚定（匹配“哪里”）

这些符号**不占用字符**，它们匹配的是“位置”（零宽度断言）。

| 语法     | 含义                                     | 实战场景                                                     |
| :------- | :--------------------------------------- | :----------------------------------------------------------- |
| **`^`**  | 匹配字符串的**开头**                     | 校验手机号必须以 1 开头：`^1...`                             |
| **`$`**  | 匹配字符串的**结尾**                     | 校验行尾不能有空格：`\s+$`                                   |
| **`\b`** | 匹配**单词边界**（单词开头或结尾的位置） | 查找完整的单词 `cat`，用 `\bcat\b`，不会误匹配 `category` 里的 `cat`。 |
| **`\B`** | 匹配**非单词边界**                       | 匹配 `category` 中的 `cat`，用 `\Bcat\B`。                   |

## 分组 `()` 与引用（制造“变量”）

括号不仅是用来“圈范围”的，它还能把匹配到的内容存起来。

1.  **捕获分组（Capturing Group）**
   - `(\d{4})-(\d{2})-(\d{2})` 匹配日期。
   - `$1` 或 `\1` 表示第一个括号匹配到的内容（年份），`$2` 是月份，`$3` 是日。
   - **实战**：替换日期格式。将 `2026-06-24` 变成 `06/24/2026`，替换为 `$2/$3/$1`。

2.  **非捕获分组 `(?:)` （只分组，不存变量）**
   - 如果你只是为了用 `|` 或量词，并不想提取内容，就用这个以提高性能。
   - `(?:http|ftp)://` 匹配协议，但不需要单独提取协议名。

3.  **反向引用（Backreference）**
   - 在正则**内部**引用前面捕获组匹配到的**具体值**（不是重复规则，是重复值）。
   - 匹配连续相同的数字：`(\d)\1{2}` 匹配 `111` 或 `222`（第一个数字是几，`\1` 就必须是几）。

## 断言 `(?=)` 与 `(?!)` （高级“透视眼”）

也叫**环视（Look around）**。它**匹配位置**，不消费字符，只是“看一眼”周围是否符合条件。

- **肯定顺序断言 `(?=...)`**：右边必须是...
  - `\d{3}(?=元)` 匹配 `100元` 中的 `100`，但不匹配 `100美元` 中的 `100`。
- **否定顺序断言 `(?!...)`**：右边必须不是...
  - `\d{3}(?!元)` 匹配 `100美元` 中的 `100`，但不匹配 `100元`。
- **肯定逆序断言 `(?<=...)`**（部分语言不支持，如 JavaScript 旧版）：左边必须是...
  - `(?<=\$)100` 匹配 `$100` 中的 `100`。
- **否定逆序断言 `(?<!)`**：左边必须不是...

## 语法优先级（决定匹配顺序）

就像四则运算的乘除优先于加减，正则也有运算优先级：

1. **转义符** `\` （最高）
2. **括号与分组** `()` `[]` 
3. **量词** `*` `+` `?` `{}` 
4. **位置锚定** `^` `$` 
5. **分支** `|` （最低）

*示例*：`^abc|def$` 匹配的是 `^abc` 或 `def$`。如果想匹配以 `abc` 或 `def` 开头的行，必须加括号：`^(abc|def)`。

## 附录：各语言“转义”大坑

正则表达式在字符串中时，编程语言会先转义一次，正则引擎再转义一次，这是**双重转义**。

- **Python**：建议使用**原始字符串** `r"\d"`，这样 `\` 不会被 Python 解释，直接交给正则引擎。
- **Java**：`String regex = "\\d";`（必须写两个 `\`，第一个转义第二个）。
- **JavaScript**：`/\d/`（字面量写法最安全）。
